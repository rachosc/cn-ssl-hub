# cn-ssl-hub

一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何单个域名或具体网站，仅作为通用 HTML 页面集中存放与版本管理的场所。

## 项目简介

`cn-ssl-hub` 旨在收集、整理并托管一系列与 SSL/TLS 相关的独立 HTML 页面。这些页面可能包含信息展示、工具界面、文档说明等，彼此之间无强关联，但均以静态 HTML 形式存在，便于直接浏览或嵌入。

仓库本身不提供任何动态服务，所有页面均为纯前端实现，无需后端支持。

## 目录结构

```
cn-ssl-hub/
├── pages/          # 存放所有独立 HTML 页面
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/         # 页面引用的公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
├── docs/           # 相关文档或说明文件（可选）
└── README.md       # 本文件
```

- `pages/`：每个 HTML 文件为一个独立页面，文件名应反映页面内容（如 `checker.html`、`guide.html`）。
- `assets/`：存放所有页面可能共享的静态资源，避免重复。
- `docs/`：用于存放额外的文档、截图、说明等（非必需）。

## 页面归档说明

- 所有页面均为静态 HTML，可离线运行。
- 页面之间无依赖关系，可单独访问。
- 归档的页面可能来自不同来源或用途，但均经过整理以确保基本可用性。
- 每个页面应包含清晰的标题和内容，避免依赖外部服务（CDN 资源除外）。

## 维护说明

- 欢迎提交 Pull Request 添加新的 HTML 页面，或对现有页面进行改进。
- 新增页面请放入 `pages/` 目录，并确保文件名无空格或特殊字符。
- 若页面需要额外的资源文件，请放入 `assets/` 对应子目录，并在页面中正确引用。
- 本仓库不保证对所有页面的长期可用性负责，归档内容仅供参考和学习。
- 如有问题或建议，请通过 Issues 反馈。
